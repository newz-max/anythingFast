# 执行、风险与日志

anythingFast 的本地系统动作统一通过 Tauri/Rust 后端执行。前端负责交互、表单、搜索和状态展示，后端负责本地动作执行、风险判断、变量解析、校验和日志记录。

## 执行方式

### 运行整个事项

运行事项时，系统会按顺序执行所有启用动作。执行前会检查事项是否启用、是否已经在运行、是否需要运行变量输入，以及是否需要风险确认。

执行完成后，事项的最近运行时间会由后端结果同步更新。

### 运行单个动作

事项详情中可以单独运行某个动作。单动作运行仍会使用事项的运行变量定义，也会执行风险分析和确认流程。

停用事项或停用动作不会被执行。

### 快捷搜索运行

快捷搜索面板运行事项时，仍属于手动执行。需要输入变量或确认风险时会弹窗，不会绕过安全流程。

从快捷面板智能创建事项时，保存会经过现有后端校验和风险归一化。启用“保存后立即执行”只是在保存成功后进入同一套手动执行流程；首次命令和高风险命令仍会要求二次确认，执行失败也不会撤销已经成功保存的事项。

### 无人值守运行

事项快捷键和周期触发属于无人值守路径。它们不能弹窗收集变量，也不能弹出高风险确认。

如果事项需要运行时输入或二次确认，无人值守触发会被阻塞，并记录执行结果。

## 风险等级

风险等级分为低、中、高。

- 低风险：打开 URL、文件、文件夹、程序或延时等待等普通动作通常为低风险。
- 中风险：普通命令动作通常为中风险。
- 高风险：脚本执行、删除、格式化、强制操作、提权、安装依赖等命令特征通常会被识别为高风险。

前端会即时显示风险，后端会在保存和执行前继续校验。不要把风险等级只当成 UI 提示。

## 高风险确认

手动执行高风险事项或高风险动作时，会弹出确认框，列出事项、动作和风险原因。用户确认后才会继续执行。

首次执行命令事项、包含高风险命令的事项，或运行时变量解析后命中高风险规则的事项，都可能需要确认。

无人值守触发无法完成确认，因此会被阻塞。

## 命令输出和日志

命令动作支持后台运行和显示终端。

- 后台运行：stdout、stderr、退出码会进入执行结果，可显示在日志中，也可通过输出绑定写入运行变量。
- 显示终端：输出主要显示在终端窗口中，执行日志只保留退出码和结果摘要。

日志会保留最近的执行摘要，用于查看成功、失败、跳过、取消、失败原因、动作耗时和命令输出片段。

## 常见阻塞场景

- 事项已停用。
- 动作已停用。
- 同一个事项或动作已经在运行。
- 必填运行变量没有值。
- 变量引用语法无效。
- 命令动作缺少工作目录。
- URL、路径、脚本类型或 Shell 组合不符合校验。
- 无人值守触发遇到需要输入变量或高风险确认的事项。
- 周期触发命中防重叠策略。

遇到阻塞时，先查看事项详情、执行日志和风险提示，再检查变量默认值、动作参数和触发器配置。
