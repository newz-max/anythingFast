# 动作配置

动作是事项真正执行的步骤。一个事项可以包含多个动作，运行时会按列表顺序执行。每个动作都有类型、参数、启用状态、风险等级，并可配置条件、超时、失败后继续和输出绑定。

## 动作类型

### 打开程序

用于启动本地可执行程序。

- 必填：程序路径。
- 可选：启动参数、工作目录。
- 适合启动 IDE、浏览器、工具软件或本地脚本宿主。

### 打开 URL

用于打开网页地址。

- URL 必须是 `http` 或 `https` 地址，除非使用运行变量占位。
- 默认使用系统浏览器行为。
- 适合常用后台、文档、项目看板和本地 Web 服务入口。

### 打开文件

使用系统默认程序打开本地文件。

- 必填：文件路径。
- 可使用运行变量拼接路径，例如 `{{ projectDir }}\README.md`。

### 打开文件夹

打开本地文件夹位置。

- 必填：文件夹路径。
- 适合项目目录、下载目录、资料目录等入口。

### 执行命令

通过本地 Shell 执行命令或脚本，是风险最高、配置项最多的动作。

- 来源：内联命令或脚本文件。
- Shell：终端默认配置、PowerShell 7、Windows PowerShell 或 cmd。
- 脚本文件：支持 `.ps1`、`.cmd`、`.bat`；`.ps1` 不能使用 cmd 执行。
- 工作目录：必填。
- 环境变量：可为命令运行过程注入额外环境变量。
- 终端显示：可后台运行，也可显示系统终端；使用“终端默认配置”时必须显示系统终端。
- 输出绑定：后台命令可把 stdout、stderr、exit code 写入运行变量，供后续动作使用。

命令动作会被识别为中风险或高风险。包含删除、格式化、强制写入、提权、安装依赖等特征时通常会触发高风险确认。

### 延时等待

在动作之间等待一段时间。

- 必填：等待时长，单位为毫秒。
- 时长必须大于 0。
- 适合等待程序启动、服务端口准备或浏览器页面加载。

## 系统集成动作

系统集成动作可与其他动作按顺序组合，用于在本地流程中交换剪贴板内容、向用户发送状态提示或等待服务就绪。

### 写入剪贴板

将文本写入系统剪贴板。

- 支持 `{{ variableKey }}` 引用运行变量。
- 动作摘要和执行日志不会回显写入内容。
- 适合将后续需要粘贴的路径、URL 或命令准备到剪贴板。

### 读取剪贴板

读取 Unicode 文本剪贴板内容，并将其写入指定的运行时变量，供后续动作和条件使用。

- 读取到的内容始终按敏感变量处理，不会显示在预览、执行事件或日志中。
- 空剪贴板会作为空字符串写入目标变量，可与“变量非空”条件组合使用。
- 该动作属于高风险动作。在主窗口或快捷搜索面板手动运行时需要二次确认。
- 事项快捷键和计划触发属于无人值守执行，不会读取剪贴板；执行会记录需要在主窗口手动运行的阻塞结果。

### 系统通知

发送系统通知，用于在流程完成、失败或等待结束时提示用户。

- 标题必填，正文可选；两者都支持 `{{ variableKey }}` 引用。
- 系统通知不可用时会记录“未发送”状态，但不会中断后续动作。

### 等待 TCP 端口

持续检查指定主机的 TCP 端口是否可以连接，连接成功后继续执行后续动作。

- 不进行 HTTP 健康检查；仅确认 TCP 连接是否成功。
- 默认超时为 60,000 ms，允许范围为 1,000-600,000 ms。
- 适合等待本地开发服务、数据库或其他依赖服务完成启动。

## 顺序、启停和失败处理

动作按列表顺序执行。停用的动作不会执行，也不会影响事项保存。

默认情况下，动作失败会导致事项失败并停止后续动作。开启“失败后继续”后，即使当前动作失败，后续动作仍可继续执行。这个选项适合非关键步骤，例如尝试打开一个可选文件或清理临时状态。

超时时间用于限制单个动作最长执行时间。超时时间必须大于 0，适合命令或脚本动作，避免长时间卡住。

## 执行条件

动作条件用于决定某个动作是否跳过：

- 始终执行。
- 仅当文件存在。
- 仅当文件夹存在。
- 仅当变量等于指定值。
- 仅当变量非空。
- 仅当上一动作成功、失败或已跳过。

条件会影响实际执行顺序和流程预览。路径条件和变量比较值中可以使用 `{{ variableKey }}` 引用运行变量。

## 流程预览

流程预览用于检查动作序列的执行结构、条件关系和运行状态。执行事项后，预览会结合执行事件显示动作的运行、成功、失败或跳过状态。

建议在保存复杂事项前检查流程预览，尤其是包含条件、输出绑定和“失败后继续”的事项。
