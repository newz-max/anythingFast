# 导入、导出与模板

anythingFast 支持把事项和模板导出为 JSON，也支持从 JSON 导入配置。导入前会显示预览，帮助你确认事项数量、模板数量、动作类型、风险摘要、路径提示和 ID 冲突处理。

## 模板来源

模板中心同时显示两类模板：

- 内置模板随应用提供，不写入用户配置，不属于用户模板导出集合。
- 已保存模板来自“保存为模板”或模板导入，会写入本地配置，可以继续使用、导入和导出。

两类模板都会生成独立、可编辑的事项草稿，不会直接执行动作，也不会自动携带快捷键或计划触发器。

模板中心会明确标记来源，并支持按场景分类或关键词查找。创建前可以预览完整动作顺序、条件、变量引用和风险；首次配置值会写入新事项的变量默认值，每次运行所需的变量仍在手动运行时填写。

## 导出事项

可以导出当前事项或当前列表中的事项。导出结果是 JSON 文件，包含事项、动作、变量、触发器、标签关联和相关元数据。

导出文件适合备份、迁移到另一台机器，或分享给其他用户作为可编辑配置。

## 导出模板

模板中心只导出已保存模板。内置模板由应用版本管理，不会混入用户模板 JSON。模板用于生成事项草稿，不会直接运行任何动作。

导出模板时，如果没有已保存模板，应用会提示没有可导出的内容。

从事项保存的模板会保留变量定义、动作顺序、输出绑定和动作条件，但不会保存可执行的触发器绑定。

## 导入配置

导入 JSON 时必须在桌面应用中使用，因为需要调用本地文件选择和后端校验能力。

导入流程：

1. 选择 JSON 文件。
2. 查看导入预览。
3. 检查事项、模板、动作数量和命令动作数量。
4. 检查风险标签和缺失路径提示。
5. 确认导入。

确认导入后，应用会用后端返回的新配置刷新前端状态。

导入的模板会作为已保存模板出现，可以再次导出。导入不会覆盖或修改随应用提供的内置模板。

## 导入预览

导入预览包含以下信息：

- 事项数量。
- 模板数量。
- 动作总数。
- 命令动作数量。
- 每个事项和模板的动作类型、动作数量和风险等级。
- 缺失路径提示。
- ID 冲突摘要。

预览不代表所有动作一定能在当前机器成功执行。它主要用于在导入前发现明显风险和路径问题。

## ID 冲突处理

如果导入文件中的事项 ID、动作 ID 或模板 ID 与当前配置冲突，系统会重生成冲突 ID。

导入预览会显示已重生成的事项 ID、动作 ID 和模板 ID 数量。这样可以避免覆盖现有事项，同时保留导入内容。

## 路径提示

导入预览会检查部分路径是否存在，并列出缺失路径提示。

常见原因：

- 导入文件来自另一台电脑。
- 文件、文件夹或程序路径不同。
- 使用了外接硬盘或网络路径。
- 路径中依赖运行变量，当前默认值不可用。

导入后建议逐个打开高风险事项和缺失路径事项，确认路径、变量和命令参数。

## 风险摘要

导入预览会标记低风险、中风险和高风险事项。包含命令、脚本或危险命令特征的事项需要重点检查。

导入不会自动运行任何动作。即使来自模板，保存和运行时仍会执行现有校验和风险确认。

使用模板创建事项时可以选择普通保存或保存并运行。保存并运行会先完成校验和持久化，再进入与事项详情相同的手动执行流程；运行变量取消、风险确认取消或执行失败都不会回滚已经保存的事项。

## 导入后的触发器

确认导入后，应用会刷新配置。包含事项快捷键或周期触发的导入内容，需要关注以下情况：

- 快捷键可能与当前全局快捷键或其他事项快捷键冲突。
- 周期触发只在应用运行时生效。
- 需要运行变量输入或高风险确认的事项无法通过快捷键或周期触发无人值守执行。

导入后建议先手动运行一次关键事项，确认变量、路径、风险和日志结果正常。
