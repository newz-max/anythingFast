# 运行变量

运行变量用于把可变输入从动作参数中抽离出来。你可以在事项中定义变量，然后在动作路径、URL、命令、脚本参数、环境变量或条件中使用 `{{ variableKey }}` 引用它。

## 变量字段

每个变量包含以下字段：

- Key：程序内部使用的变量名，例如 `projectDir`。
- 标签：运行前输入窗口中显示给用户看的名称，例如“项目目录”。
- 默认值：没有运行时输入时使用的值。
- 必填：运行前必须提供非空值。
- 敏感：输入框按密码形式显示，日志和文本展示中会尽量遮罩真实值。

## Key 命名规则

Key 只能包含字母、数字和下划线，且不能以数字开头。

有效示例：

```text
projectDir
branch_name
_token
apiKey2
```

无效示例：

```text
1token
project-dir
project.dir
中文变量
```

同一个事项内变量 Key 不能重复，标签不能为空。

## 引用语法

在支持变量的文本参数中使用双花括号引用：

```text
{{ projectDir }}
{{branchName}}
```

花括号内部可以有空格。引用的 Key 必须符合变量命名规则。出现未闭合的 `{{` 或多余的 `}}` 会被视为无效语法。

## 支持引用的位置

以下位置支持运行变量引用：

- 打开程序：程序路径、工作目录、启动参数。
- 打开 URL：URL、浏览器字段。
- 打开文件：文件路径。
- 打开文件夹：文件夹路径。
- 执行命令：命令内容、工作目录、脚本路径、脚本参数、环境变量值。
- 执行条件：文件路径、文件夹路径、变量等于条件中的比较值。

示例：

```text
{{ projectDir }}\README.md
git -C "{{ projectDir }}" checkout {{ branchName }}
http://localhost:{{ port }}
```

## 默认值和运行前输入

运行事项时，系统会先基于默认值生成变量值。

如果变量是必填，或没有默认值，手动执行时会弹出“填写运行变量”窗口。用户确认后，运行时输入会覆盖默认值。

如果变量既不是必填，又有默认值，手动执行时不会询问，直接使用默认值。

## 敏感变量

敏感变量适合保存 token、密码、密钥等内容。

- 默认值和运行前输入会用密码输入框显示。
- 执行日志和错误文本会尽量把敏感变量值替换为 `••••`。
- 不建议把敏感信息写进事项名称、动作名称或普通描述，因为这些字段不是密钥存储。

## 命令输出绑定

命令动作可以把执行结果写回运行变量上下文，供后续动作使用：

- stdout 绑定：把标准输出写入指定变量。
- stderr 绑定：把错误输出写入指定变量。
- exit code 绑定：把退出码写入指定变量。

输出绑定只影响本次运行的变量上下文，不会改写事项中变量的默认值。

输出绑定变量可以不是事先定义的输入变量。只要 Key 有效，后续动作就可以引用它：

```text
命令动作 A：把 stdout 绑定到 currentBranch
命令动作 B：echo 当前分支是 {{ currentBranch }}
```

输出绑定只对“执行命令”动作生效。命令后台运行时更适合使用输出绑定；显示终端时，用户主要在终端窗口里查看输出。

## 与快捷键和周期触发的关系

事项快捷键和周期触发属于无人值守执行路径，不能弹出运行变量输入窗口，也不能弹出高风险二次确认。

因此，以下事项不能通过事项快捷键或周期触发静默执行：

- 存在必填变量且没有默认值。
- 存在没有默认值的变量。
- 执行前需要高风险二次确认。

这类触发会被阻塞并记录结果。需要无人值守执行时，请为变量提供默认值，并避免需要二次确认的高风险动作。
