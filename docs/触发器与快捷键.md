# 触发器与快捷键

事项可以手动运行，也可以通过系统级事项快捷键或周期触发运行。应用还提供全局唤起快捷键和软件内快捷键，用于提升操作效率。

## 手动触发

每个事项默认都有手动触发。手动触发会完整执行交互流程：

- 停用事项不会运行。
- 需要运行变量时会弹出输入窗口。
- 高风险事项会弹出二次确认。
- 执行结果会写入执行日志。

快捷搜索面板中的 `Enter` 执行，以及智能创建的“保存后立即执行”，都属于手动执行流程。

## 全局唤起快捷键

默认全局快捷键是 `Alt+Space`，用于唤起或隐藏快捷搜索面板。

全局唤起快捷键是系统级快捷键，可以在设置中修改。修改时需要处理系统注册状态：如果快捷键被其他程序占用，应用会显示不可用状态。

## 快捷搜索面板快捷键

快捷搜索面板常用快捷键：

- `ArrowUp` / `ArrowDown`：选择搜索结果。
- `Enter`：有搜索结果时执行选中事项；没有结果且存在可保存的智能创建建议时打开确认表单。
- `Escape`：确认表单打开时取消确认，否则关闭面板。
- `/`：聚焦搜索框。
- `Ctrl+N`：打开主窗口的新增事项流程。

搜索会匹配事项名称、分类、描述、关键词、动作名称、动作类型和动作描述。没有输入搜索词时，快捷搜索面板会优先展示最近运行过的事项。

当没有已有事项匹配时，HTTP(S) URL、已有 Windows 文件/文件夹路径和非空 `cmd:` 命令可以显示智能创建建议。已有事项结果始终优先，不会与创建建议混排。

## 事项快捷键

事项快捷键用于直接触发指定事项，适合稳定、无需交互的事项。

注意事项：

- 事项快捷键是系统级快捷键，不是软件内快捷键。
- 事项快捷键不能与全局唤起快捷键冲突。
- 不同事项的快捷键不能重复。
- 如果事项停用，快捷键不会执行它。
- 如果事项需要运行变量输入或高风险二次确认，快捷键触发会被阻塞并记录结果。

建议只给低风险、参数固定、可以无人值守执行的事项配置快捷键。

## 软件内快捷键

软件内快捷键只在对应窗口或编辑器聚焦时生效，例如主窗口、快捷搜索、事项编辑器和动作编辑器。

它们与系统级快捷键是不同层级。软件内快捷键可以用于搜索、运行选中事项、新增事项、编辑事项、保存和关闭编辑器等常用操作。

## 周期触发

周期触发用于在应用运行时自动执行事项。当前支持三种模式：

- 每隔一段时间：间隔不能小于 5 分钟。
- 每天固定时间：时间格式为 `HH:mm`。
- 每周固定时间：需要至少选择一天，并设置 `HH:mm` 时间。

周期触发的下次运行时间会在保存后计算。周期触发只在应用运行时生效。

## 错过执行策略

周期触发支持两种错过执行策略：

- 错过则跳过：如果应用未运行或错过时间，不补跑。
- 恢复后补跑一次：恢复后补跑一次错过的执行。

具体行为还会受到事项状态、运行变量、高风险确认和防重叠策略影响。

## 防重叠策略

周期触发支持防重叠控制：

- 运行中跳过新触发：同一事项仍在运行时，新的周期触发会跳过。
- 允许重叠执行：允许同一事项被多次周期触发并发运行。

对会修改文件、执行命令或调用外部系统的事项，建议保持“运行中跳过新触发”。
