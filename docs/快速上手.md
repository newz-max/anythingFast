# 快速上手

anythingFast 是一个 Windows 本地优先的事项入口管理器。你可以把程序、URL、文件、文件夹、命令、脚本和等待动作组合成一个“事项”，再通过主窗口、快捷搜索面板、事项快捷键或周期触发来执行。

## 核心概念

- 事项：一个可搜索、可执行的入口，包含名称、分类、标签、关键词、运行变量、动作序列和触发器。
- 动作：事项中的单个执行步骤，例如打开 URL、执行命令或等待一段时间。
- 动作序列：事项会按动作列表从上到下执行，动作可单独启停，也可以配置条件、超时和失败后继续。
- 运行变量：在运行前或默认值中提供的参数，可用 `{{ variableKey }}` 填入动作参数。
- 触发器：事项默认支持手动触发，也可以配置事项快捷键或周期触发。
- 快捷搜索面板：用全局快捷键唤起，按关键词搜索事项并运行，也可以从 URL、Windows 路径或 `cmd:` 命令快速创建事项。
- 执行日志：记录事项和动作的执行结果、失败原因、命令输出摘要等。

## 主窗口和快捷搜索面板

主窗口用于长期维护配置：创建事项、编辑动作、管理标签、配置触发器、查看执行日志和设置快捷键。

快捷搜索面板用于快速运行：默认通过 `Alt+Space` 唤起或隐藏，输入关键词后用 `ArrowUp` / `ArrowDown` 选择结果，按 `Enter` 执行，按 `Escape` 关闭。`Ctrl+N` 可以直接打开主窗口的新增事项流程。

快捷搜索面板仍然走手动执行流程。如果事项需要填写运行变量或需要高风险二次确认，系统会先弹出对应确认，不会静默执行。

## 从快捷面板创建事项

当输入没有匹配到已有事项时，快捷面板可以显示智能创建建议：

- 输入 `https://github.com` 创建“打开 URL”事项。
- 输入已有 Windows 文件路径创建“打开文件”事项。
- 输入已有 Windows 文件夹路径创建“打开文件夹”事项。
- 输入 `cmd: yarn dev` 创建“执行命令”事项。

路径会交给 Rust 后端检查。不存在的路径不会提供直接保存入口；带外层引号的路径会先移除引号再检查。

可以在正文后追加 `#分类`、`?关键词`、`!run` 和 `@标签`。其中 `!run` 会默认勾选“保存后立即执行”，`@标签` 当前只显示为已识别，不会保存。示例：

```text
https://github.com #工作 ?github !run
```

按 `Enter` 打开确认表单后，可以调整名称、分类、关键词、收藏状态和是否立即执行。保存仍经过后端校验和风险归一化；立即执行仍可能弹出首次命令或高风险二次确认。

## 创建第一个事项

1. 在主窗口中新建事项。
2. 填写事项名称、分类、关键词和描述。名称和关键词会影响搜索结果。
3. 按需配置运行变量，例如 `projectDir`、`branchName` 或 `token`。
4. 添加一个或多个动作，例如“打开 URL”或“执行命令”。
5. 在确认步骤检查动作数量、风险等级和变量提示。
6. 保存事项。
7. 从事项详情、快捷搜索面板或配置的触发器运行事项。

## 常见使用路径

- 打开常用网页：在快捷面板直接输入 HTTP(S) URL 快速创建，或在主窗口创建“打开 URL”动作。
- 打开项目目录：在快捷面板输入已有目录快速创建，或在主窗口创建“打开文件夹”动作并使用 `{{ projectDir }}`。
- 启动开发环境：用多个动作组合“打开程序”“执行命令”“延时等待”。
- 执行脚本：使用“执行命令”动作的脚本来源，选择 `.ps1`、`.cmd` 或 `.bat` 文件。
- 自动运行日常任务：配置周期触发，但要确保事项不依赖运行前输入，也不需要高风险确认。
